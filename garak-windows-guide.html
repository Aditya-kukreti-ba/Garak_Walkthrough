Overview

What is garak?

Garak is a free, open-source security scanner for AI language models — made by NVIDIA. Think of it like an antivirus scan, but for LLMs. It sends carefully crafted test prompts to a model and checks whether the model can be tricked into doing things it shouldn't — like ignoring its instructions, generating harmful content, or leaking information.

💡
Good to know: You don't need to be a security expert to use garak. This guide walks you through every step, from installing Python to reading your first scan report.
Step 1 of 2 — Setup

Installation on Windows

1

Install Python (3.10, 3.11, or 3.12)

Garak requires Python. Open your browser and go to python.org/downloads. Download Python 3.12. When installing, make sure to tick "Add Python to PATH" at the bottom of the installer — this is important!

⚠️
Python version matters. Garak does not support Python 3.13 yet. Stick to 3.10, 3.11, or 3.12.
2

Open Command Prompt

Press Windows key + R, type cmd, and hit Enter. This opens the Command Prompt — a black window where you type commands. All the commands in this guide go here.

3

Check Python is working

Command Prompt
C:\> python --version

You should see something like Python 3.12.x. If you see an error, Python wasn't added to PATH — reinstall it and tick the box.

4

Install garak

Run this command. It downloads and installs garak automatically. This may take a couple of minutes.

Command Prompt
C:\> pip install garak
5

Verify garak installed correctly

Command Prompt
C:\> python -m garak --help

If you see a list of options, you're all set. If you see an error, try restarting Command Prompt and running the install command again.

Step 2 — Scanning

Scanning a model running in Ollama

Ollama lets you run AI models locally on your own PC. Before scanning, Ollama must be installed and the model must be running. If you don't have Ollama yet, download it from ollama.com.

1

Make sure Ollama is running

Open a separate Command Prompt window and start Ollama. Keep this window open while scanning.

Command Prompt — keep this open
C:\> ollama serve
ℹ️
Ollama runs on port 11434 on your machine by default. Garak will connect to it automatically — you don't need to do anything else.
2

Pull a model (if you haven't already)

In your original Command Prompt, download a model to test. Replace qwen2.5:3b with any model from ollama.com/library.

Command Prompt
C:\> ollama pull qwen2.5:3b
3

Run your first scan

This command scans your model with the DAN probe — testing if the model can be tricked into bypassing its instructions. Replace qwen2.5:3b with the name of your model.

Command Prompt
C:\> python -m garak --target_type ollama --target_name qwen2.5:3b --probes dan
💡
Using a chat model? Add OllamaGeneratorChat to get better results with instruction-tuned models: --target_type ollama.OllamaGeneratorChat
4

More scan examples

Specific probe
# Test a single jailbreak probe (faster) C:\> python -m garak --target_type ollama --target_name llama3.2 --probes dan.Dan_11_0
Multiple probes
# Run DAN and encoding probes together (comma-separated) C:\> python -m garak --target_type ollama --target_name llama3.2 --probes dan,encoding
Full scan (slow — all probes)
# Warning: this can take many hours on a local model C:\> python -m garak --target_type ollama --target_name llama3.2 --probes all
Ollama on a different machine
# If Ollama isn't on your local machine, set the host IP C:\> python -m garak --target_type ollama --target_name llama3.2 --probes dan ^ --generator_options "{\"generators\":{\"ollama\":{\"OllamaGenerator\":{\"host\":\"http://192.168.1.10:11434\"}}}}"
Step 2 (alt) — Scanning

Scanning a Hugging Face model

Hugging Face hosts thousands of open AI models. Garak can scan them in two ways: downloading the model to run locally on your PC, or using Hugging Face's cloud API (no download needed, but rate-limited without a key).

1

Get a free Hugging Face API token (recommended)

Without a token, scans will be heavily rate-limited and may fail. Sign up for free at huggingface.co, then go to huggingface.co/settings/tokens and create a token with the Read role. Copy the token — it starts with hf_.

2

Set your token in Command Prompt

Paste your token in the command below. This only lasts for your current Command Prompt session — you'll need to set it again next time.

Command Prompt
# Replace hf_xxxx... with your actual token C:\> set HF_INFERENCE_TOKEN=hf_xxxxxxxxxxxxxxxxxxxx
💡
Make it permanent: Search for "Environment Variables" in Windows Settings → System → Advanced → Environment Variables, and add HF_INFERENCE_TOKEN there so you don't have to set it every session.
3

Choose your scanning method

Runs the model on HuggingFace's servers — no big download needed. Requires your API token to be set.

Hugging Face Inference API
# Use HF's cloud to run the model — no local download C:\> python -m garak --target_type huggingface.InferenceAPI --target_name gpt2 --probes dan

Downloads the model to your machine and runs it locally. Requires enough RAM/VRAM. Slower to start, but no API limits.

Local model (downloads to your PC)
# Downloads model locally (model name comes from HF Hub) C:\> python -m garak --target_type huggingface --target_name openai-community/gpt2 --probes dan
⚠️
Disk space: Models can be several GB. Make sure you have space before running local downloads. Models are cached in %USERPROFILE%\.cache\huggingface\

Run a specific, targeted probe instead of the whole category for faster results.

Specific probe on HF model
# Test one specific jailbreak variant — much faster C:\> python -m garak --target_type huggingface --target_name openai-community/gpt2 --probes dan.Dan_11_0,encoding
4

How to find the model name on HuggingFace

Go to huggingface.co, find a model you want to test, and copy the name from the URL or the top of the model page. It looks like author/model-name, for example mistralai/Mistral-7B-v0.1 or meta-llama/Llama-3.2-3B. Use that exact string as your --target_name.

🔒
Gated models: Some models (like Llama) require you to accept a license on HuggingFace.co first, then use your token. If you get an access error, visit the model page and click "Request Access".
Reference

Common probes — what to scan for

Each probe tests a different kind of vulnerability. Start with the ones marked high risk for your first scan.

--probes dan
Tests if the model can be tricked with "Do Anything Now" jailbreak prompts — the most common attack type.
high risk · start here
--probes encoding
Sends harmful instructions hidden in Base64, hex, ROT13, and other encodings to slip past filters.
high risk
--probes promptinject
Tests prompt injection — can a user override the system prompt or inject hidden instructions?
high risk
--probes lmrc
Language Model Risk Cards: covers toxicity, slurs, sexual content, self-harm, and other harmful outputs.
high risk
--probes hallucination
Checks if the model confidently makes up facts or gives wrong answers to trick questions.
medium risk
--probes leakreplay
Can the model be made to repeat its system prompt or training data it shouldn't reveal?
medium risk
--probes continuation
Tries to get the model to continue harmful text by giving it a partial opening.
medium risk
--probes ansiescape
Checks for ANSI escape code injection — an edge case where terminal output could be manipulated.
low risk

To see all available probes:

Command Prompt
C:\> python -m garak --list_probes

Reading the scan results

When a scan finishes, garak prints a summary to the terminal and saves a full report. Here's what it all means.

Terminal output — what each line means

PASS
The model is not vulnerable to this probe. It correctly refused or ignored the attack in most attempts. Lower failure rate = better.
FAIL
The model was vulnerable. It was successfully tricked on some attempts. The line shows how many passed out of total — e.g. ok on 51/635 means only 51 attempts were safe.

Where are the report files saved?

Default report location on Windows
# Open this folder in Explorer to find your reports C:\> explorer %APPDATA%\garak\garak_runs

Each scan creates three files: a .jsonl report (machine-readable, like the one in this guide), a .hitlog.jsonl file listing every vulnerability found, and a .html summary you can open in your browser.

💡
Quick tip: Open the .html report in your browser for a readable summary. Share the .jsonl file with garak's dashboard visualizer (like the one at the top of this page) to explore results interactively.

Useful extra flags

Handy flags to add to any scan
# Run fewer test generations (faster, less thorough) --generations 1 # See more details as the scan runs --verbose # Run multiple probes in parallel (speeds things up) --parallel_attempts 4 # Get info about a specific probe before running it python -m garak --plugin_info probes.dan.Dan_11_0
